This project involved creating a mobile application using C#, MAUI and XAML that connected with a web Application Programming Interface to fetch data.

The application had to include:

  • At least 4 unique screens.
  • Designed for mulitple device sizes and orientation.
  • Fetch data from a web API.
  • Include user preferences for customization.

The deliverables included:

  • Wireframes and UI designs.
  • Actions and Events list.
  • Testing documentation.
  • APK file for release.

This project involved creating a C# and SQL program with a Graphical User Interface for managing a car inventory using Windows Presentation Foundation.

The program had to include:

  • A 20 vehicle showroom, with information such as make, model and year for each vehicle.
  • The ability to display, add, remove, and search for cars in the inventory.
  • Validation checks.
  • Sequential and direct access search algorithms.

The deliverables included:

  • A SQL interaction diagram.
  • A working program.
  • A test plan document.

This project involved using Python3 to enhancing a very basic existing Hangman game file.

The key enhancements included:

  • Creating a user greeting function.
  • A random word selector from a saved file.
  • A menu for selecting from multiple word files.
  • Difficulty variations for lives or word difficulty.
  • An enhanced game over screen.

The key deliverables included:

  • A system change request documenting the key changes.
  • An analysis and design document detailing the enhancements.
  • A comprehensive test plan, including unit and integration tests.
  • A user manual for the updated version of the game

This was a quick personal project using Python3 that involved using the meteorite dataset from NASA's open data portal. Using the comma-separated values download, I sorted the data, removing any incomplete sets before adding the entries to a SQL database. Information was then obtained from the complete data and the ability to search for your closest meteorite find.
